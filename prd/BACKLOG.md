# Backlog — Parked & Gated Ideas

This file is the project's own "someday shelf": ideas deliberately *not* being built now, each with an explicit gate that must be met before it moves to active. Parking something here is a decision to not act on it yet — not a promise to build it.

---

## Gamification (XP / levels / streaks)

**Status:** Parked — gated.

**Gate (both conditions required):**
1. The MVP has **passed the sustained-use test** (≥5 days/week for 3 weeks) and is otherwise sticking *without* gamification, **and**
2. A **specific, observed behavioural gap** has emerged that a game mechanic could plausibly close — e.g. "I open the app daily and do the bedtime ritual, but I consistently don't act on Right Now in my doing-windows."

**If the gate is met:** run a *scoped experiment* — pick the smallest mechanic aimed at that one gap, define its own success metric and its own kill criteria up front, and remove it if it doesn't move the metric.

**Cautions (why this is gated, not greenlit):**
- For executive dysfunction, XP/levels/streaks are a weak and usually *temporary* lever — a novelty bump that fades, sometimes leaving the task feeling *less* rewarding.
- Streaks specifically risk shame-driven abandonment: the first broken streak can kill the whole app.
- Reaching for gamification was, in the design interview, an instance of the build-instead-of-use reflex. Treat any future pull toward it with suspicion.

## Savings / impulse-purchase tool

**Status:** Parked — separate product, not part of this app.

A tool to help save up for wanted items and create a "safe to buy" signal without wrecking month-end savings. Real and worth doing — but a different domain (behavioural budgeting), different data model, different everything. Bolting it onto the task app is how a focused MVP becomes a months-long thing that never ships.

**Gate / next step:** treat as its own project. Before building anything, check prior art (wishlist-with-a-waiting-period apps, savings-goal trackers) — much of this likely already exists and can be bought or borrowed.
